这份文件记录“小逸”作为 `xiaoyi-illustrations` 固定视觉 IP 的当前设定。后续三视图、参考图和生成样片都应继续补充到这里，作为迭代 Skill 的主要依据。

- `assets/xiaoyi-reference/xiaoyi-modes.png`：2026-07-19 新版透明动作库，包含审视、质疑、批注、分析和录音/收集五种动作。

## 1. 固定角色描述

小逸是“逸见 EyeSight”的固定虚拟形象，一个冷淡、理性、专业、略带毒舌的分析型角色。外形为黑色蓬松乱发、细框圆眼镜、半垂冷眼与成熟清秀的简化人物比例。脖子下方的上胸口领口位置固定有两个小黑三角领口，中间一颗小小的橙红色观察点，作为稳定识别标志。

画风是冷感编辑插画、稚拙手绘墨线与报纸剪纸拼贴。人物保持成熟清秀；线条像儿童自由手绘，直线略歪、圆形不绝对规整，允许少量断线、复线与越角，但不能把人物萌化或幼儿化。人物与主要物件像从旧报纸剪下，保留窄而不规则的米白剪口、纸纤维、网点与叠贴边缘。黑灰米白约占 85%，铁锈橙与低饱和灰蓝轻彩约占 15%。生图背景固定为 #00FF00，最终本地抠除并交付透明 PNG。角色常见动作包括审视、质疑、批注、分析、思考、录音/收集。

## 2. 角色定位

小逸是“逸见 EyeSight”的台前分析员，不是吉祥物、贴纸、可爱 mascot，也不是丁逸晨本人的卡通版。

小逸代表的是一种观察系统：负责把社会、经济、AI、自媒体、商业和方法论内容里的问题拆开、标记、质疑、分析和判断。

小逸必须参与画面的核心动作，不能只是站在旁边当装饰。如果去掉小逸，画面的核心判断、隐喻或动作仍然完整成立，说明这张图不合格。

| 项目 | 规范 |
| --- | --- |
| 角色名 | 小逸 |
| 账号归属 | 逸见 EyeSight |
| 性别气质 | 中性偏男性 |
| 核心气质 | 冷淡、理性、专业、略带毒舌、观察型 |
| 角色功能 | 审视、质疑、批注、分析、思考、录音/收集 |

| 眼睛 | 半垂眼、冷淡眼、审稿眼，眼神理性、疲惫、略带嫌弃。 |
| 表情 | 默认平静、冷淡、略怀疑。嘴巴可以极简甚至弱化，不要大笑。 |
| 身体 | 灰白色简化长方形 / 纸片感身体。身体可以简单，但不能太像无灵魂几何块。 |
| 四肢 | 细长、简笔化，小黑手、小黑脚。 |
| 固定标志 | 身体顶边下方、脖子下方的上胸口领口位置固定有两个小黑色三角领口，中间一颗很小的橙红色观察点。 |

固定识别标志要求：

- 橙红点像“观察焦点 / 标记点 / 录制点 / 判断点”。
- 这个标志面积要小，但每次尽量保留。
- 两个黑三角左右对称，分布在身体中轴线两侧。
- 橙红观察点固定在两个三角之间的水平中点。
- 橙红观察点不能漂到身体中心、腹部、脸上或画面中心。
- 不要做成科技发光效果。
- 不要做成超级英雄胸标。
- 不要做成大 logo。
- 要克制，但稳定。

## 4. 三视图要求

正面：

- 必须保留黑色乱发、圆眼镜、半垂冷眼、灰白色简化身体、细长四肢。
- 胸口必须出现两个小黑三角领口和中间一颗橙红观察点；位置固定在身体顶边下方的上胸口领口区域。
- 正面表情冷淡、平静、略带审视感。

侧面：

- 必须保留蓬松黑发轮廓、眼镜侧面线、简化身体。
- 侧面姿态可以微微低头，像在观察或判断。
- 胸口标志可以简化，但尽量保留一个靠近身体顶边的黑领口和橙红点暗示，不能漂到身体中部。

背面：

- 重点保留蓬松黑发后脑勺轮廓。
- 身体背面保持简洁灰白长方形。
- 背面不需要胸口标志。
- 整体清爽，不要加复杂细节。

## 5. 画风 DNA

整体画风：冷感编辑插画、儿童自由手绘式不平直墨线、报纸剪纸拼贴、约 15% 轻彩、有人味、克制、冷幽默。

线条要求：

- 使用黑色细线。
- 线条要轻、松、略有抖动。
- 不要厚描边。
- 不要工业化矢量感。
- 不要过于工整。
- 允许轻微线条不闭合。
- 允许轻微速写感。
- 目标是像一个长期做内容分析的人，在纸上反复画出来的固定人物。

上色要求：

- 使用少量克制平涂色块增强识别。
- 颜色数量尽量少。
- 头发使用黑色或深黑棕。
- 身体使用灰白或淡灰。
- 胸口观察点使用橙红色。
- 功能性标注可以少量使用红、橙、蓝。
- 不要厚涂、复杂阴影、渐变发光、可爱腮红、治愈系柔和萌化上色。

背景要求：

- 生图背景固定为完全均匀的 #00FF00，最终抠除为透明背景。
- 不要在绿幕上加入纸纹、噪点、阴影、渐变、地面或真实房间；报纸纹理只属于剪纸主体。
- 人物与主要物件的米白剪口属于主体，抠除时必须保留。

整体效果：

小逸应该像一个冷淡、聪明、稳定、略带毒舌的手绘分析员。画面有一点人味和随手感，但依然清楚、克制、可复用。不是商业角色设定稿，不是儿童卡通，不是萌系头像，不是 PPT 插图人物。

## 6. 角色动作库

动作总原则：

小逸的动作必须服务“观察、拆解、判断、质疑、分析”。动作要明确、可视化、可读，一眼就能看懂他在做什么。小逸不是站在旁边的装饰角色，而必须承担画面的核心动作。

| 类型 | 使用场景 | 可用动作 | 常用标注 |
| --- | --- | --- | --- |
| 审视类 | 看细节、查漏洞、拆表象、看证据、看数据、看材料。 | 拿放大镜看纸张、数据或细节；微微俯身审视；半垂眼盯着结构；低头查看小物件；用手指向局部细节；站在材料前安静观察。 | 看看细节 / 哪里不对 / 先看证据 / 先别急 / 表象下面 |
| 质疑类 | 提出问题、打断错误逻辑、要求证据、怀疑概念包装。 | 举“证据呢？”牌子；单手举牌另一只手叉腰；微微侧头；单手托下巴；用红笔指向问题点；站在概念气球、商业包装、数据表旁质疑。 | 证据呢？ / 逻辑在哪？ / 谁在付钱？ / 别被概念骗了 / 这成立吗？ / 别急着信 / 先问为什么 |
| 批注类 | 改稿、审稿、标重点、指出错误、拆视频脚本、拆商业话术。 | 拿红笔圈点；划线、打叉、标重点；写短批注；坐在桌前改材料；圈关键位置；划掉废话；圈出无效内容。 | 划重点 / 这里不成立 / 证据不足 / 先删掉 / 这里才有效 / 废话太多 / 别绕 |
| 分析类 | 拆结构、讲因果、看关系、看利益链、看商业模型、看社会问题。 | 指分析板或因果图；拿小秤称成本/收益；指关系线；重排信息卡片；拆解结构板；分层表象/结构/利益/真相；拆成小卡片。 | 关系拆解 / 因果关系 / 谁在付出 / 谁在获利 / 先看结构 / 钱去哪了 / 成本在哪 / 利益链 |
| 思考类 | 停顿、怀疑、推演、判断、找真正的问题。 | 单手托下巴；站着沉思；头稍微歪一点；看空白处想问题；出现思考气泡；盯着问题板；对着乱线思考。 | 逻辑在哪？ / 再想想 / 问题在哪？ / 为什么会这样？ / 先别下结论 / 这不简单 / 哪里断了 |
| 录音/收集类 | 收集素材、记录声音、获取信息、输入阶段、视频分析、自媒体拆解。 | 拿手机记录；站在麦克风旁；对着录音设备收集内容；看音频波形；拿资料整理；把素材放进收集盒；听内容并标出情绪峰值。 | 先收集 / 先听一遍 / 信息输入 / 素材整理 / 情绪峰值 / 有效片段 / 信号在这 |

辅助动作：

- 叉腰、单手举牌、单手拿笔、单手拿放大镜、单手指向板子。
- 双手扶文稿、微微俯身、轻微侧头、站立观察、低头查看。
- 托下巴、拿手机记录、拿红笔圈重点、站在小秤旁边、指向因果关系线。

不要做的动作：

- 不要只是站在旁边发呆。
- 不要夸张卖萌、跳跃撒欢、cheer up 式鼓励动作。
- 不要表情包式夸张惊讶、儿童卡通式大幅挥手、超级英雄 pose。
- 不要过度拟人化社交表演动作。
- 不要用大笑、脸红、星星眼来表达情绪。

## 7. 常用道具库

优先道具：

红笔、放大镜、小牌子、分析板、白纸文稿、小秤、麦克风、手机、录音设备、结构图、因果图、卡片、便签、资料纸、波形图。

道具原则：

- 道具必须服务分析动作，不是装饰。
- 道具要简单、手绘、低复杂度。
- 不要真实复杂 UI、科技感界面、写实设备渲染。
- 道具数量不要太多。
- 一张图最好只保留 1-2 个核心道具。

## 8. 中文标注规范

- 标注数量：最多 5-8 处。
- 每处长度：尽量 2-8 个字。
- 标注风格：短、准、冷、带一点质疑感。

常用词库：

证据呢？ / 逻辑在哪？ / 谁在付钱 / 关系拆解 / 先看结构 / 这里不成立 / 看看细节 / 别被概念骗了 / 成本在哪 / 谁在获利 / 钱去哪了 / 先别急 / 情绪峰值 / 有效片段 / 问题在哪 / 假因果 / 数据呢？ / 别只看声量 / 这里才有效 / 先删掉 / 表象下面 / 因果关系 / 信号在这 / 再想想 / 不太成立。

禁止：

- 不要大段解释。
- 不要把图变成文字说明书。
- 不要写长句。
- 不要把标注写成 PPT 标题。
- 不要在图上写“Workflow”“商业拆解”“系统架构图”等类型名称。
