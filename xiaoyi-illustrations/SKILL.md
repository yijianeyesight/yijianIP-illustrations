# 小逸冷脸正文配图

## 核心定位

为中文文章设计和生成 16:9 横版正文配图。目标不是做商业插画、PPT 信息图或可爱卡通，而是把内容里的关键判断、结构漏洞、利益关系、流程、状态或隐喻，变成一张清爽、怪诞、有创意、可读但不说明书的手绘解释图。

默认视觉 IP 是“小逸”：黑色蓬松乱发、细框圆眼镜、半垂冷眼、成熟清秀的简化人物比例、上胸口两个小黑三角领口与其间一颗很小的橙红观察点。小逸像一位认真参与社会、经济、AI、自媒体和商业问题拆解的冷脸分析员，必须参与画面的核心判断或操作动作，不能只是站在旁边当装饰。

默认画风是“冷感编辑插画 × 稚拙手绘线 × 报纸剪纸拼贴”：人物保持成熟好看，但线条像儿童自由手绘那样不追求机械平直；人物与不同物件像从旧报纸剪下，保留窄而不规则的米白剪口、纸纤维、网点和叠贴边缘。不要把“儿童手绘线条”误解为儿童角色、可爱比例或儿童绘本。

## 工作流

### 1. 消化正文

先读用户给的正文、链接、Notion 页面、Markdown 文件或截图内容。提炼：

- 核心观点是什么
- 哪些段落承担认知转折
- 哪些内容适合用图解释
- 哪些地方只适合文字，不需要图

不要平均配图。优先选择“认知锚点”，例如：核心判断、结构漏洞、利益关系、输入输出闭环、分流、前后对比、成本转移、证据链、假因果、承接路径、常见坑、角色状态变化。

### 2. 先出配图策略

如果用户只是说“分析怎么配图 / 思考哪些地方需要配图”，先给 shot list。每张图写清楚：

- 放在哪个段落后
- 图的主题
- 核心意思
- 结构类型
- 小逸在图里做什么
- 建议元素
- 建议中文标注词

默认 4-8 张。文章很短时 1-3 张；长文也不要轻易超过 9 张。够用就好，避免把正文做成画册。

### 3. 单张生成

如果用户明确要求“生成 / 输出 / 做图 / 帮我生成”，不要停下来等确认；用内置 `image_gen` 每张单独生成。不要把多张图拼在一张里。

每张图只讲一个核心结构。提示词必须包含：

- 16:9 横版中文正文配图
- 最终交付为透明 PNG；生图阶段使用均匀 `#00FF00` 绿幕
- 稚拙、轻微抖动、不追求平直的黑色手绘线；允许复线、断线和越过拐角
- 人物和主要物件各自带窄而不规则的米白报纸剪口、纸纤维、网点和叠贴边缘
- 黑灰为主，铁锈橙与低饱和灰蓝轻彩合计约占 15%
- 默认不写标题或说明文字；确需标注时保持极少、极短
- 大量留白
- 小逸作为核心判断或操作动作主体
- 小逸的关键识别锚点：黑色蓬松乱发、细框圆眼镜、半垂冷眼、成熟清秀的人物比例、上胸口两个小黑三角领口、一颗位于两个三角之间的很小橙红观察点
- 禁止 PPT、商业插画、幼稚可爱、日系动漫化、完美矢量直线、复杂架构和左上角类型标题

不要复刻过往案例。案例只提供风格密度和角色参与方式，不能直接复用旧构图，除非用户明确要求复刻某张图。每次都要从当前文章重新发明一个奇怪但成立的隐喻。

### 4. 检查与迭代

生成后检查 `references/qa-checklist.md`。如果出现以下问题，优先重生成或局部编辑：

- 小逸只是装饰
- 小逸缺少圆眼镜、黑色乱发、半垂冷眼、灰白长方形身体、小黑手脚、上胸口领口三角或两个三角之间的橙红观察点
- 画面太满
- 太像流程图/PPT
- 中文太多或错字严重
- 左上角出现“常见坑/流程图/系统架构图”等标题
- 画风太可爱、幼稚、死板
- 背景不是干净白底

### 5. 透明 PNG 处理

默认把最终正文插图交付为透明 PNG：

1. 生图时把外围背景固定为完全均匀的 `#00FF00`；保留人物与物件自身的米白剪纸边。
2. 使用 `scripts/remove_chroma_key.py` 本地抠除绿幕，不依赖手工选区。
3. 默认命令、参数和透明通道验收见 `references/transparent-output.md`。
4. 同时保留绿幕源图和透明 PNG，不覆盖原始生成文件。
5. 若头发、纸纤维或剪纸白边出现绿边，先用 `--despill`；仍有细边时再用 `--edge-contract 1`，不要直接加大容差吞掉白边。

### 6. 保存交付

如果用户在 workspace 内工作，把最终图复制到：

```text
assets/<article-slug>-illustrations/
```

按顺序命名：

```text
01-topic-name.png
02-topic-name.png
```

保留原始生成文件，不要覆盖已有资产，除非用户明确要求替换。

## 输出口径

生成前的策略输出要短而准。生成后的交付要包含：

- 生成了几张
- 每张图的用途
- 保存路径
- 哪些图最稳，哪些图是可选

不要长篇解释风格理论；让图自己说话。
