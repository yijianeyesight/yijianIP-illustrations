# 风格 DNA

## 一句话

纯白、轻手绘、细线、简笔、随手感、有人味、克制、冷幽默、观察型、中文手写感。

像一个长期做内容分析、AI、商业拆解和社会观察的人，在纸上反复画出来的固定人物和解释草图。看起来轻松，但判断很准。

## 必须

- 16:9 横版正文配图。
- 纯白背景。
- 黑色细线，线条要轻、松、略有抖动。
- 不要厚描边，不要工业化矢量感，不要过于工整。
- 允许轻微线条不闭合，允许轻微速写感。
- 小逸要像冷淡、聪明、稳定、略带毒舌的手绘分析员。
- 画面有一点人味和随手感，但依然清楚、克制、可复用。
- 可以加入少量克制的平涂上色增强识别度。
- 小逸身体使用灰白或淡灰，头发使用黑色或深黑棕。
- 胸口观察点使用很小的橙红色。
- 大量留白：主体占画面约 40%-60%，至少 35% 空白，画面要有呼吸感。
- 少量中文手写批注：最多 5-8 处，每处尽量 2-8 个字。
- 文字要短、准、冷、带一点质疑感。
- 一张图只讲一个核心判断、结构、状态或隐喻。
- 结构要自然表达，不要在图上写结构类型名称。

## 角色识别锚点

- 黑色蓬松乱发，略散，有自然翘起。
- 细框圆眼镜。
- 半垂冷眼、审稿眼。
- 灰白色简化长方形 / 纸片感身体。
- 细长四肢，小黑手小黑脚。
- 身体顶边下方、脖子下方的上胸口领口位置固定有两个小黑三角领口。
- 两个黑色三角领口左右对称，分布在身体中轴线两侧。
- 两个黑色三角领口之间的水平中点固定有一颗很小的橙红观察点。
- 橙红观察点不能漂到身体中心、腹部、脸上或画面中心。

## 颜色

- 黑色：线稿、头发、眼镜线、角色轮廓、工具、主要文字、小黑手脚、胸口三角领口。
- 灰白 / 淡灰：小逸身体主色。
- 橙红色：胸口观察点、少量判断点或标记点。
- 红色：毒舌批注、问题、证据质问、风险提醒、关键判断，例如“证据呢”“别被概念骗了”“假因果”。
- 橙色：主流程、路径、流向、注意力路径、转化路径。
- 蓝色：补充说明、系统状态、AI/工具状态、内容分析标注、第二层解释，例如“情绪峰值”“上下文”“波形”。

颜色要克制，宁可少不要多。默认黑白灰为主，红橙蓝只做少量功能性标注。

## 背景

- 纯白背景。
- 不要米色纸张背景。
- 不要复古纸纹。
- 不要噪点。
- 不要阴影。
- 不要渐变。
- 不要真实房间背景。

## 绝对不要

- 不要商业插画。
- 不要 PPT 信息图。
- 不要正式流程图。
- 不要课程课件。
- 不要儿童插画。
- 不要可爱卡通海报。
- 不要日系萌角色。
- 不要表情包。
- 不要真人肖像漫画。
- 不要复杂架构图。
- 不要精致扁平插画。
- 不要科技感 UI。
- 不要真实 App 截图。
- 不要复杂背景、渐变、阴影、纹理。
- 不要厚涂。
- 不要复杂阴影。
- 不要渐变发光。
- 不要可爱腮红。
- 不要治愈系柔和萌化上色。
- 不要把每个节点都解释清楚。
- 不要左上角写“Workflow 流程图 / 系统架构图 / 常见坑 / 路线图 / 商业拆解”等类型标题。

## 审美方向

要冷淡、聪明、克制、观察型、有一点人味、有轻微毒舌和冷幽默。

不要可爱、治愈、热情、幼稚、很复杂、死板、油画感、精致商业感。
